# 智慧宠物管理系统 - 快速开始指南

## 📋 前置准备

在开始之前，请确保你的系统已安装以下软件：

### 必需软件
- ✅ **JDK 1.8+** - Java开发环境
- ✅ **Maven 3.6+** - Java项目构建工具
- ✅ **MySQL 8.0+** - 数据库
- ✅ **Node.js 16+** - JavaScript运行环境

### 可选软件
- ⭕ **Redis 5.0+** - 缓存服务（可选，不影响基本功能）

## 🚀 快速启动（推荐）

### Windows用户

1. **初始化数据库**（首次运行）
   ```cmd
   cd database
   init_database.bat
   ```
   - 按提示输入MySQL密码（默认：root）
   - 等待数据库初始化完成

2. **启动项目**
   ```cmd
   start.bat
   ```
   - 脚本会自动检查环境
   - 自动编译后端项目
   - 自动安装前端依赖
   - 启动前后端服务

3. **访问系统**
   - 前端地址：http://localhost:3000
   - 后端地址：http://localhost:8080/api
   - 默认账号：admin / admin123

4. **停止服务**
   ```cmd
   stop.bat
   ```

### Linux/Mac用户

1. **添加执行权限**
   ```bash
   chmod +x start.sh stop.sh database/init_database.sh
   ```

2. **初始化数据库**（首次运行）
   ```bash
   cd database
   ./init_database.sh
   ```

3. **启动项目**
   ```bash
   ./start.sh
   ```

4. **访问系统**
   - 前端地址：http://localhost:3000
   - 后端地址：http://localhost:8080/api
   - 默认账号：admin / admin123

5. **停止服务**
   ```bash
   ./stop.sh
   ```
   或按 `Ctrl+C` 停止

## 🔧 手动启动

如果自动脚本遇到问题，可以手动启动：

### 1. 数据库初始化

```bash
# 登录MySQL
mysql -u root -p

# 执行SQL脚本
source database/smart_pet_system.sql

# 或者直接导入
mysql -u root -p < database/smart_pet_system.sql
```

### 2. 配置修改

编辑 `backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/smart_pet_system?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root        # 修改为你的MySQL用户名
    password: root        # 修改为你的MySQL密码
  
  redis:
    host: localhost       # 修改为你的Redis地址
    port: 6379
    password:             # 如果有密码，填写Redis密码
```

### 3. 启动后端

```bash
cd backend
mvn clean install
mvn spring-boot:run
```

等待看到以下信息表示启动成功：
```
========================================
智慧宠物管理系统启动成功！
访问地址：http://localhost:8080/api
========================================
```

### 4. 启动前端

打开新的终端窗口：

```bash
cd frontend
npm install
npm run dev
```

看到以下信息表示启动成功：
```
VITE v4.x.x  ready in xxx ms

➜  Local:   http://localhost:3000/
➜  Network: use --host to expose
```

## ❓ 常见问题

### 1. 数据库初始化失败

**问题**：运行 init_database.bat 时出现密码错误
```
ERROR 1045 (28000): Access denied for user 'root'@'localhost'
```

**解决方案**：

**方法1：使用简化版脚本（推荐）**
```cmd
cd database
# 用记事本打开 init_simple.bat
# 修改第10行: set MYSQL_PASSWORD=你的实际密码
# 保存后运行
init_simple.bat
```

**方法2：手动初始化**
- 查看 `database/手动初始化数据库.md` 详细指南
- 使用MySQL Workbench图形化工具导入SQL
- 或使用命令行：`mysql -u root -p < smart_pet_system.sql`

### 2. 数据库连接失败

**问题**：启动后端时提示数据库连接失败

**解决方案**：
- 检查MySQL服务是否启动
- 确认数据库配置信息是否正确
- 确认数据库 `smart_pet_system` 是否已创建

### 3. 端口被占用

**问题**：启动时提示端口已被占用

**解决方案**：

Windows:
```cmd
# 查看占用8080端口的进程
netstat -ano | findstr :8080
# 结束进程（PID为查询到的进程ID）
taskkill /F /PID <PID>
```

Linux/Mac:
```bash
# 查看占用8080端口的进程
lsof -i :8080
# 结束进程
kill -9 <PID>
```

### 4. 前端依赖安装失败

**问题**：npm install 失败

**解决方案**：
```bash
# 清除npm缓存
npm cache clean --force

# 使用淘宝镜像
npm install --registry=https://registry.npmmirror.com

# 或使用cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install
```

### 5. Maven编译失败

**问题**：mvn clean install 失败

**解决方案**：
```bash
# 清除Maven本地仓库缓存
mvn clean

# 使用阿里云镜像
# 编辑 ~/.m2/settings.xml，添加阿里云镜像配置
```

### 6. Redis连接失败

**问题**：启动时提示Redis连接失败

**解决方案**：
- Redis是可选的，不影响基本功能
- 如果不使用Redis，可以注释掉相关配置
- 或者安装并启动Redis服务

## 📞 技术支持

如果遇到其他问题，请检查：
