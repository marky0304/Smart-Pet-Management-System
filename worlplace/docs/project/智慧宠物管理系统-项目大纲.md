# 智慧宠物管理系统 - 项目大纲

## 一、项目概述

### 1.1 项目名称
智慧宠物管理系统 (Smart Pet Management System)

### 1.2 技术栈
- **后端**: Spring Boot 2.7+, MyBatis-Plus, MySQL 8.0, Redis
- **前端**: Vue 3, Element Plus, Axios, Vue Router
- **开发工具**: Maven, Node.js, Git

### 1.3 系统简介
智慧宠物管理系统是一个面向宠物主人和管理员的综合性平台，旨在提供宠物信息管理、健康监测、预约服务、社区交流等功能，帮助用户更好地照顾宠物，同时为宠物服务机构提供高效的管理工具。

## 二、系统实现目标

### 2.1 业务目标
1. **提升宠物照护质量**: 通过系统化管理宠物信息、健康记录，帮助主人更科学地照顾宠物
2. **优化服务流程**: 实现在线预约、服务管理，提高宠物服务机构的运营效率
3. **促进社区互动**: 建立宠物主人交流平台，分享养宠经验和知识
4. **数据驱动决策**: 通过数据统计分析，为管理员提供运营决策支持

### 2.2 技术目标
1. **前后端分离架构**: 采用RESTful API设计，实现前后端解耦
2. **高性能**: 使用Redis缓存热点数据，优化数据库查询
3. **安全性**: 实现JWT身份认证、权限控制、数据加密
4. **可扩展性**: 模块化设计，便于功能扩展和维护
5. **用户体验**: 响应式设计，支持PC端和移动端访问

### 2.3 用户目标
1. **普通用户**: 便捷管理宠物信息，获取专业服务，参与社区交流
2. **管理员**: 高效管理系统数据，监控运营状态，提供优质服务

## 三、用户角色定义

### 3.1 普通用户 (User)
- 注册登录系统
- 管理个人宠物信息
- 查看和预约宠物服务
- 记录宠物健康数据
- 参与社区交流
- 查看个人订单和消费记录

### 3.2 管理员 (Admin)
- 系统用户管理
- 宠物信息审核与管理
- 服务项目管理
- 预约订单管理
- 社区内容审核
- 数据统计与分析
- 系统配置管理

## 四、系统功能模块

### 4.1 用户管理模块
**功能描述**: 负责用户注册、登录、个人信息管理和权限控制

**主要功能**:
- 用户注册与登录（支持手机号/邮箱）
- 个人信息管理（头像、昵称、联系方式）
- 密码修改与找回
- 用户权限管理（普通用户/管理员）
- 用户列表查询与管理（管理员）
- 用户状态管理（启用/禁用）

### 4.2 宠物档案管理模块
**功能描述**: 管理宠物的基本信息和档案资料

**主要功能**:
- 宠物信息录入（名称、品种、年龄、性别、照片）
- 宠物档案查看与编辑
- 宠物疫苗接种记录
- 宠物过敏史与特殊说明
- 宠物列表展示（支持筛选和搜索）
- 宠物信息审核（管理员）
- 宠物统计分析（品种分布、年龄分布）

### 4.3 健康管理模块
**功能描述**: 记录和追踪宠物的健康状况

**主要功能**:
- 健康记录添加（体重、体温、症状）
- 就医记录管理（医院、诊断、处方）
- 用药提醒设置
- 健康趋势图表展示
- 健康报告生成
- 异常健康数据预警
- 健康数据统计分析（管理员）

### 4.4 服务预约模块
**功能描述**: 提供宠物相关服务的在线预约功能

**主要功能**:
- 服务项目展示（洗澡、美容、寄养、医疗）
- 服务详情查看（价格、时长、说明）
- 在线预约（选择时间、宠物、服务）
- 预约订单管理（查看、取消、评价）
- 服务项目管理（管理员：增删改查）
- 预约审核与处理（管理员）
- 服务统计报表（管理员）

### 4.5 商城管理模块
**功能描述**: 提供宠物用品的在线购买功能

**主要功能**:
- 商品分类展示（食品、玩具、用品、药品）
- 商品详情查看（图片、价格、库存、描述）
- 购物车管理
- 订单创建与支付
- 订单查询与物流跟踪
- 商品管理（管理员：上下架、库存管理）
- 订单管理（管理员：发货、退款处理）
- 销售数据统计（管理员）

### 4.6 社区交流模块
**功能描述**: 为宠物主人提供经验分享和交流的平台

**主要功能**:
- 发布动态（文字、图片、话题标签）
- 浏览动态列表（热门、最新、关注）
- 点赞、评论、转发
- 话题广场（热门话题、话题分类） 
- 用户关注与粉丝
- 内容搜索功能
- 内容审核与管理（管理员）
- 违规内容处理（管理员）

### 4.7 数据统计模块
**功能描述**: 为管理员提供系统运营数据的统计和分析

**主要功能**:
- 用户数据统计（注册量、活跃度、增长趋势）
- 宠物数据统计（数量、品种分布、年龄分布）
- 服务数据统计（预约量、服务类型分布、收入统计）
- 商城数据统计（销售额、热销商品、库存预警）
- 社区数据统计（发帖量、互动数据、热门内容）
- 数据可视化图表（折线图、柱状图、饼图）
- 数据导出功能（Excel、PDF）

## 五、系统架构设计

### 5.1 后端架构
```
smart-pet-system
├── common (公共模块)
│   ├── utils (工具类)
│   ├── config (配置类)
│   └── exception (异常处理)
├── entity (实体类)
├── mapper (数据访问层)
├── service (业务逻辑层)
├── controller (控制器层)
└── security (安全认证)
```

### 5.2 前端架构
```
pet-management-web
├── src
│   ├── api (接口请求)
│   ├── assets (静态资源)
│   ├── components (公共组件)
│   ├── views (页面视图)
│   ├── router (路由配置)
│   ├── store (状态管理)
│   └── utils (工具函数)
```

### 5.3 数据库设计
主要数据表：
- user (用户表)
- pet (宠物表)
- health_record (健康记录表)
- service (服务项目表)
- appointment (预约订单表)
- product (商品表)
- order (订单表)
- post (动态表)
- comment (评论表)

## 七、非功能性需求

### 7.1 性能要求
- 页面响应时间 < 2秒
- 支持1000+并发用户
- 数据库查询优化

### 7.2 安全要求
- 密码加密存储
- JWT Token认证
- SQL注入防护
- XSS攻击防护
- 敏感数据加密

### 7.3 可用性要求
- 系统可用性 > 99%
- 定期数据备份
- 异常日志记录
- 错误提示友好

### 7.4 兼容性要求
- 支持Chrome、Firefox、Safari、Edge浏览器
- 响应式设计，适配PC和移动端
- 支持主流分辨率
