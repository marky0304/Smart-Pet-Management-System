### 后端技术栈

- **Spring Boot 2.7.14** - 基础框架
- **MyBatis-Plus 3.5.3.1** - ORM框架
- **MySQL 8.0** - 数据库
- **Redis** - 缓存
- **JWT** - 身份认证
- **Hutool** - 工具库
- **Lombok** - 简化代码

### 后端分层说明

1. **Controller层** - 接收HTTP请求，调用Service层处理业务
2. **Service层** - 业务逻辑处理
3. **Mapper层** - 数据访问，与数据库交互
4. **Entity层** - 实体类，对应数据库表
5. **DTO层** - 数据传输对象，用于接收前端请求
6. **VO层** - 视图对象，用于返回给前端

## 🔄 数据流向

```
前端 Vue 组件
    ↓ (用户操作)
前端 API 调用 (axios)
    ↓ (HTTP请求)
后端 Controller (接收请求)
    ↓ (参数校验)
后端 Service (业务处理)
    ↓ (数据操作)
后端 Mapper (数据访问)
    ↓ (SQL执行)
MySQL 数据库
    ↓ (返回数据)
后端 Service (数据处理)
    ↓ (封装响应)
后端 Controller (返回结果)
    ↓ (HTTP响应)
前端接收数据
    ↓ (状态更新)
前端页面渲染
```

## 🔐 认证流程

```
1. 用户登录
   ↓
2. 后端验证用户名密码
   ↓
3. 生成JWT Token
   ↓
4. 返回Token给前端
   ↓
5. 前端存储Token到localStorage
   ↓
6. 后续请求携带Token
   ↓
7. 后端拦截器验证Token
   ↓
8. Token有效则放行，无效则返回401
```

## 📝 开发规范

### 后端开发规范

1. **命名规范**
   - 类名：大驼峰（UserController）
   - 方法名：小驼峰（getUserInfo）
   - 常量：全大写下划线（MAX_SIZE）

2. **包结构规范**
   - controller：控制器
   - service：业务逻辑
   - mapper：数据访问
   - entity：实体类
   - dto：请求对象
   - vo：响应对象

3. **接口规范**
   - RESTful风格
   - 统一返回Result对象
   - 统一异常处理

### 前端开发规范

1. **命名规范**
   - 组件名：大驼峰（UserProfile.vue）
   - 方法名：小驼峰（getUserInfo）
   - 常量：全大写下划线（API_BASE_URL）

2. **目录规范**
   - views：页面组件
   - components：公共组件
   - api：接口调用
   - utils：工具函数

3. **代码规范**
   - 使用Vue 3 Composition API
   - 使用setup语法糖
   - 统一使用Pinia管理状态

## 🚀 扩展开发

### 添加新功能模块

1. **后端开发步骤**
   - 创建Entity实体类
   - 创建Mapper接口
   - 创建Service接口和实现
   - 创建Controller控制器
   - 编写DTO和VO类

2. **前端开发步骤**
   - 在api/目录创建接口文件
   - 在views/目录创建页面组件
   - 在router/index.js添加路由
   - 在layout中添加菜单项

### 数据库变更

1. 修改database/smart_pet_system.sql
2. 重新执行初始化脚本
3. 或使用数据库迁移工具
