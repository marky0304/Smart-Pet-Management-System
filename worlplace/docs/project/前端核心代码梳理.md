# 智慧宠物管理系统 — 前端核心代码梳理

## 项目概览

| 项目 | 说明 |
|------|------|

## 二、核心文件详解

### 2.1 应用入口 — main.js

- 创建 Vue 应用
- 注册插件链: VueRouter → Pinia → ElementPlus(完整引入) → 所有 ElementPlus 图标
- 挂载到 `#app`

### 2.2 Vite 配置 — vite.config.js

```javascript
export default defineConfig({
  plugins: [vue()],
  resolve: { alias: { '@': '/src' } },
  server: {
    port: 3000,
    proxy: { '/api': { target: 'http://localhost:8080', changeOrigin: true } }
  }
})
```

```

  if (路由需 token && 无 token) → 重定向 /login
  else → 正常放行
```

### 2.4 状态管理 — store/user.js

```javascript
state: {
  token: localStorage.getItem('token') || '',
  userInfo: JSON.parse(localStorage.getItem('userInfo') || '{}')
}

getters: {
  isLogin → !!token,
  isAdmin → userInfo.role === 'ADMIN'
}

actions: {
  login(username, password) → authAPI.login() → 存 token/userInfo 到 localStorage,
  logout() → 清除 localStorage + reset state
}
```

### 2.5 Axios 封装 — utils/request.js

```javascript
const service = axios.create({ baseURL: '/api', timeout: 10000 })

// 请求拦截器: 自动附加 Authorization: Bearer <token>
// 响应拦截器: code !== 200 弹出错误提示; 401 清除 token 跳转登录
```

> 注意: 部分 API 文件（chat.js, pet.js, community.js, user.js）各自创建了独立 Axios 实例，存在重复代码。

## 三、智能客服组件（重点）

### 3.1 ChatBubble.vue — 浮动按钮

- 右下角固定 56×56px 蓝色渐变圆形按钮
- 点击切换聊天窗口开/关
- 显示未读消息角标 (unreadCount)，打开窗口时清零
- 弹跳 + 波纹扩散 CSS 动画

### 3.2 ChatWidget.vue — 聊天窗口

```
功能:
  - 380×520px 绝对定位窗口，在 ChatBubble 上方
  - 头部: "PetCare 智能客服" + 关闭按钮
  - 消息列表: 自动滚底，区分用户/AI 消息气泡
  - 三个快捷按钮: 健康查询 / 商品推荐 / 饲养建议
  - 输入框 + 发送按钮（Enter 发送）
  - 显示 Agent 标签: 标识回复来自哪个 AI Agent

  每次发送携带:
    - sessionId: 会话标识（多轮对话）
    - currentPage: 当前路由路径（上下文感知）

  executeAction() 解析返回的 action JSON:
    - PAGE_NAVIGATE 类型 → 自动 router.push() 页面跳转
    - 其他类型 → 渲染确认信息
```

**数据流**:
```
用户输入 → chatAPI.sendMessage(message, sessionId, currentPage)
        → POST /api/chat/send
        → { reply, operation, domain, agentType, navigate, action, sessionId }
        → 渲染 AI 回复 (含 Agent 标签)
        → 有 navigate → router.push() 跳转页面
```

### 3.3 ChatConfig.vue — 客服设置页 (管理员)

**4 个 Tab**:

| Tab | 配置项 | 控件 |
|-----|--------|------|
| 服务商 | 服务商选择、API 地址、API Key、模型名称 | 下拉框 + 输入框 |
| 模型参数 | Temperature 0~2、Max Tokens 100~8000 | 滑块 |
| 聊天行为 | 启用/关闭开关、自定义欢迎语 | 开关 + 文本域 |
| Prompt 定制 | 自定义系统提示词 | 文本域 |

**操作按钮**: "保存设置" (PUT /api/chat-config) + "测试连接" (发送测试请求验证 API)

## 五、关键业务流程

### 5.1 认证流程

```
Login.vue → authAPI.login({username, password})
         → 获取 {token, userInfo}
         → 存入 localStorage + Pinia store
         → router.push('/dashboard/home')
         → 后续所有 API 请求自动携带 Authorization: Bearer <token>
         → 401 响应 → 清除登录态 → 重定向 /login
```

### 5.2 商城购物流程

```
Shop.vue
  → 商品列表 (分类筛选/搜索/排序/分页)
  → 商品详情弹窗
  → 加购: cartAPI.addToCart(productId, quantity)
  → Cart.vue
    → 购物车列表 (多选/数量调整/删除)
    → 结算: 选地址 → 选支付方式 → 填备注 → 提交
    → orderAPI.createOrder({items, addressId, ...})
    → 跳转 我的订单
```

### 5.3 健康管理流程

```
Health.vue
  → 筛选: 宠物下拉 / 记录类型 / 日期范围
  → ECharts 可视化:
    - 饼图: 记录类型分布
    - 柱状图: 月度健康记录统计
    - 折线图: 体重/体温趋势
  → 数据表格: 健康记录列表 (CRUD)
```

### 5.4 管理员配置 LLM 流程

```
ChatConfig.vue
  → Tab 服务商: 选服务商 / 填 API URL / API Key / 模型名
  → Tab 模型参数: Temperature 滑块 / Max Tokens 滑块
  → Tab 聊天行为: 启用开关 / 欢迎语
  → Tab Prompt 定制: 自定义系统提示词
  → "保存设置" → chatConfigAPI.batchSave(configs)
  → "测试连接" → 验证 API 可用性
```

## 六、设计特点

1. **JWT 认证**: localStorage 持久化 token，Axios 拦截器自动携带
2. **路由守卫**: 未登录重定向，基于 token 的前端鉴权
3. **购物车双模式**: 登录用户调后端 API，未登录用户 localStorage 兜底
4. **智能客服懒加载**: ChatWidget 使用 `defineAsyncComponent` 按需加载
5. **数据可视化**: 健康管理使用 ECharts 饼图/柱状图/折线图
6. **管理员页面**: 7 个管理页面 + 1 个客服设置页面，导航菜单条件渲染
7. **CSS 变量体系**: 全局设计 Token（品牌色/阴影/圆角/间距）统一管理
