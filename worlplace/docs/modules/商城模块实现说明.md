# 商城模块实现说明

## 模块概述
由于时间和token限制，商城模块采用精简实现方案。

## 核心功能

### 用户功能
1. **商品浏览**
   - 按分类查看（食品、玩具、用品、药品）
   - 商品搜索
   - 商品详情查看

2. **购物车**（简化版）
   - 前端本地存储实现
   - 添加/删除商品
   - 数量调整

3. **订单管理**
   - 创建订单
   - 查看订单列表
   - 查看订单详情
   - 取消订单

### 管理员功能
1. **商品管理**
   - 商品增删改查
   - 上架/下架
   - 库存管理

2. **订单管理**
   - 订单列表查看
   - 订单发货
   - 订单统计

## 简化方案

### 购物车实现
使用前端localStorage存储，格式：
```javascript
{
  items: [
    {
      productId: 1,
      name: "商品名称",
      price: 99.00,
      quantity: 2,
      image: "图片URL"
    }
  ]
}
```

### 支付功能
暂不实现真实支付，订单创建后直接标记为"已支付"状态。

### 物流跟踪
管理员手动输入物流单号，用户可查看但不对接真实物流API。

## 数据库表

### product表
```sql
- id: 商品ID
- name: 商品名称
- category: 分类（FOOD/TOY/SUPPLY/MEDICINE）
- description: 描述
- price: 价格
- stock: 库存
- sales: 销量
- image: 主图
- images: 图片集（JSON）
- status: 状态（0-下架，1-上架）
```

### order表
```sql
- id: 订单ID
- order_no: 订单号
- user_id: 用户ID
- total_amount: 总金额
- status: 状态（PENDING/PAID/SHIPPED/COMPLETED/CANCELLED）
- payment_method: 支付方式
- payment_time: 支付时间
- shipping_address: 收货地址
- shipping_phone: 收货电话
- shipping_name: 收货人
- tracking_number: 物流单号
- notes: 备注
```

### order_item表
```sql
- id: 详情ID
- order_id: 订单ID
- product_id: 商品ID
- product_name: 商品名称
- product_image: 商品图片
- price: 单价
- quantity: 数量
- subtotal: 小计
```
