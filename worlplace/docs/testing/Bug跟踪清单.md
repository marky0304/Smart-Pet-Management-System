**问题描述**:
Service实体类名与Spring的@Service注解冲突，导致Maven编译失败。

1. 运行 `mvn clean compile`
2. 编译失败，提示类型冲突

```
com.pet.entity.Service的 single-type-import 已定义具有相同简名的类型
```

实体类名Service与Spring框架的@Service注解冲突。

**修复方案**:
1. 将Service.java重命名为PetService.java
2. 更新ServiceMapper中的引用
3. 更新ServiceServiceImpl中的所有引用
4. 更新AppointmentServiceImpl中的引用

**验证结果**: ✅ 通过
- Maven编译成功
- 后端服务正常启动
- 功能测试正常

## Bug优先级定义

### 🔴 高优先级（P0）
- 系统崩溃或无法启动
- 核心功能完全不可用
- 数据丢失或损坏
- 安全漏洞
- 影响所有用户

**处理时间**: 立即修复（24小时内）

### 🟡 中优先级（P1）
- 重要功能异常
- 影响用户体验
- 性能问题
- 影响部分用户

**处理时间**: 3个工作日内

### 🟢 低优先级（P2）
- UI显示问题
- 文字错误
- 非核心功能异常
- 优化建议

**处理时间**: 下个版本修复

## Bug处理流程

```
发现Bug → 记录Bug → 分析原因 → 分配任务 → 修复Bug → 代码审查 → 测试验证 → 关闭Bug
```

### 1. 发现Bug
- 测试人员发现
- 用户反馈
- 开发自测发现

### 2. 记录Bug
- 填写Bug模板
- 添加到Bug清单
- 设置优先级

### 3. 分析原因
- 重现问题
- 分析日志
- 定位代码

### 4. 分配任务
- 根据模块分配
- 根据优先级排序
- 设置截止日期

### 5. 修复Bug
- 编写修复代码
- 添加单元测试
- 更新文档

### 6. 代码审查
- 提交Pull Request
- 代码审查
- 修改建议

### 7. 测试验证
- 功能测试
- 回归测试
- 性能测试

### 8. 关闭Bug
- 更新Bug状态
- 记录修复方案
- 归档文档

## 常见问题FAQ

### Q1: 如何报告Bug？
A: 使用Bug模板填写详细信息，包括重现步骤、预期结果、实际结果等。

### Q2: Bug优先级如何确定？
A: 根据影响范围、严重程度、用户数量等因素综合判断。

### Q3: 修复后如何验证？
A: 按照重现步骤进行测试，确保问题已解决且没有引入新问题。

### Q4: Bug修复后需要做什么？
A: 更新Bug状态、记录修复方案、进行回归测试、更新文档。
