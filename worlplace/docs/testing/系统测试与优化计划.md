## 二、性能优化计划

### 2.1 数据库优化

#### 索引优化
```sql
-- 检查现有索引
SHOW INDEX FROM user;
SHOW INDEX FROM pet;
SHOW INDEX FROM health_record;
SHOW INDEX FROM appointment;
SHOW INDEX FROM product;
SHOW INDEX FROM `order`;
SHOW INDEX FROM post;

-- 添加必要的复合索引
-- 用户表：按角色和状态查询
CREATE INDEX idx_role_status ON user(role, status);

-- 宠物表：按用户和类型查询
CREATE INDEX idx_user_type ON pet(user_id, type);

-- 健康记录：按宠物和日期查询
CREATE INDEX idx_pet_date ON health_record(pet_id, record_date);

-- 预约表：按用户和状态查询
CREATE INDEX idx_user_status ON appointment(user_id, status);

-- 订单表：按用户和状态查询
CREATE INDEX idx_user_status ON `order`(user_id, status);

-- 动态表：按创建时间查询（热门动态）
CREATE INDEX idx_create_time_status ON post(create_time DESC, status);
```

#### 查询优化
- [ ] 使用EXPLAIN分析慢查询
- [ ] 避免SELECT *，只查询需要的字段
- [ ] 使用JOIN代替子查询
- [ ] 合理使用分页
- [ ] 添加查询缓存

#### 数据库配置优化
```ini
# my.cnf 优化建议
[mysqld]
# 连接数
max_connections = 200

# 缓冲池大小（根据服务器内存调整）
innodb_buffer_pool_size = 1G

# 查询缓存
query_cache_size = 64M
query_cache_type = 1

# 慢查询日志
slow_query_log = 1
slow_query_log_file = /var/log/mysql/slow.log
long_query_time = 2
```
